# portfolio_asukaooi
ご提示いただいた内容は、ポートフォリオや成果物として非常に評価が高くなる素晴らしいポイントばかりです。技術的な選定理由や工夫（PRGパターンや防御的プログラミングなど）が具体的であるため、これらがしっかりと伝わるREADMEに仕上げましょう。
マークダウンとしてそのまま貼り付けて使える、詳細な構成案を作成しました。
🤖 自動販売機シミュレーター (Vending Machine Simulator)
Webブラウザ上で動作する、自動販売機のシミュレーターアプリケーションです。
💡 プロジェクト概要
本プロジェクトは、自動販売機の「消費者」としての購入体験と、「業者」としての管理業務をWeb上で再現したシステムです。
👥 2つのロール（役割）構成
消費者側: お金の投入、商品の選択、商品の購入、釣り銭の回収など、一般的な自販機の一連の動作を行えます。
業者側: 専用の管理画面から、商品の補充・入れ替え、価格設定、売上金の回収などの管理業務を行えます。
🛠️ フレームワーク（Spring等）を使用せず、Servlet/JSPで実装した理由
モダンなフレームワーク（Spring Boot等）に依存せず、Java Web開発の標準仕様である Jakarta Servlet / JSP を用いてネイティブに実装しました。
その理由は、フレームワークが裏側で自動処理している「HTTPリクエスト/レスポンスの仕組み」「セッション管理」「MVCパターンの本質」をブラックボックス化せずに深く理解し、堅牢な基礎技術力を証明するためです。
🛠 使用技術 (Tech Stack)
言語: Java 17
Web技術: Jakarta Servlet 5.0 / JSP 3.0 / JSTL 2.0
Webサーバー: Apache Tomcat 10.1.x
データ管理: セッションスコープ（メモリ内擬似データベース / ※DB接続がある場合はここに記載）
🖥 画面構成と機能一覧
🛍 1. 消費者側（メイン画面）
お金投入機能: 現金（10円、50円、100円、500円、1000円札）の投入処理
購入判定機能: 投入金額に応じて、購入可能な商品のボタンを活性化
商品購入機能: 在庫がある商品を選択し、購入する処理
釣り銭出力機能: 購入後、または「返却」ボタン押下時に、釣り銭を計算して返却する処理
⚙ 2. 業者側（管理画面）
認証機能: 管理セッションを用いた、業者専用画面へのアクセス制限
商品交換・カスタム機能: 商品名、価格、画像、初期在庫数の変更・補充
売上回収機能: 自販機内部に蓄積された売上金額の確認および回収（ゼロリセット）処理

✨ 工夫した点・こだわり（技術的アプローチ）
PRG（Post-Redirect-Get）パターンの適用
商品購入やお金の投入など、状態を変更する処理（POST）の後は必ずリダイレクト（GET）を挟む設計にしています。これにより、ユーザーがブラウザで「更新（F5）」ボタンを連打しても、二重送信（二重購入や二重チャージ）が発生しないように防御しています。
適切なスコープ管理（Session vs Request）
自販機全体の在庫状態や業者のログイン状態、現在の投入金額は保持し続ける必要があるため「セッションスコープ」で管理。一方で、1回限りの購入成否メッセージやエラーメッセージは「リクエストスコープ」に格納し、画面表示後に即時破棄されるよう制御しました。
サーバーサイドでの状態管理（クライアント改ざん防止）
「現在の投入金額」や「商品価格」などの重要な数値は、JavaScript等のクライアント側ではなく、すべてサーバー側のセッション内で管理・計算しています。これにより、開発者ツール等を用いた不正な金額改ざんによる購入を根本的に防いでいます。
防御的プログラミングの徹底
リクエストパラメータの null チェックや空文字チェックを網羅。数値変換時の NumberFormatException や、在庫不足時の独自例外（例: OutOfStockException）など、例外の種類を適切に使い分けることで、システムが異常終了（画面の白化や500エラー）せず、ユーザーに適切なエラーメッセージを返せる堅牢な設計にしました。
🚀 起動方法
📋 前提条件
Eclipse (Java EE 開発に対応したもの、もしくはエンタープライズ版)
Java 17 がインストールされ、Eclipseに登録されていること
🔧 設定手順
プロジェクトのインポート
Eclipseを開き、ファイル ＞ インポート ＞ 既存のプロジェクトをワークスペースへ から本リポジトリを選択して完了します。
Tomcat 10 の設定
Eclipseの「サーバー」ビューで右クリック ＞ 新規 ＞ サーバー を選択。
Apache Tomcat v10.1 を選択し、Tomcatのインストールディレクトリを指定します。
プロジェクトの追加
作成したTomcatサーバーを右クリック ＞ 追加および削除 から、本プロジェクトを「構成済み」に追加します。
起動
サーバーを右クリック ＞ 開始 をクリックします。
🌐 起動URL
ブラウザを開き、以下のURLにアクセスしてください。

http://localhost:8080/プロジェクト名/ (※環境に合わせてコンテキストパスを変更してください)
